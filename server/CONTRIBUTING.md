# Contributing

Thanks for helping improve Sanctum.

## Getting started

1. Fork or clone the **[cxl-sanctum](../README.md)** monorepo and follow [README.md](README.md) in this directory to run migrations, the API, and tests (`python manage.py test core`).
2. Make focused changes with clear commit messages.
3. Run tests before opening a pull request.

## Pull requests

- Describe **what** changed and **why** (behavior, not only filenames).
- Link related issues or discussions when applicable.
- Keep diffs scoped to the task; avoid unrelated refactors.

## Code style

Match existing patterns in the Django apps (`core`, etc.): imports, naming, and DRF usage.

## Community

Be respectful and assume good intent. For security-sensitive reports, use [SECURITY.md](SECURITY.md) instead of public issues.
